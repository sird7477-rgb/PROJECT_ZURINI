# [자동매매 시스템 시퀀스 다이어그램 - Extreme Simplicity]

## 1. Startup & Beta Throttling (08:30)
이진법적 사고를 버리고 매크로 위험을 선형적으로 수용하는 비중 조절 시작 시퀀스입니다.

```mermaid
sequenceDiagram
    participant ME as Main Engine
    participant API as Broker API
    participant RM as Risk Manager

    ME->>API: 1. Get Nasdaq 100 Futures Return
    alt API Success
        API-->>ME: Return Rate (e.g., -0.8%)
        ME->>RM: Calculate Multiplier (M)
        RM-->>ME: M = 0.84 (84% Sizing Mode)
    else API Fail / Timeout
        ME->>RM: Default Safety Sizing
        RM-->>ME: M = 0.50 (Conservative Half-Mode)
    end
    ME->>ME: Set TRADING_MODE with M
```

## 2. Fast Entry Sequence (Async NLP Blacklist)
지연의 근원인 실시간 NLP를 제거하고, 오직 DB 조회만으로 진입 속도를 극대화했습니다.

```mermaid
sequenceDiagram
    participant ME as Main Engine
    participant BL as Blacklist DB (Async NLP Output)
    participant DB as Atomic DB

    ME->>ME: Tier 1: Candidate Selection
    ME->>BL: Check Stock ID & Heartbeat (Timestamp)
    
    alt Heartbeat > 5min (Stale Data)
        BL-->>ME: Fail-Safe Triggered
        ME->>ME: Halt New Entry (System Warning)
    else Heartbeat OK & Found in Blacklist
        BL-->>ME: BAD (Risk Detected)
        ME->>ME: Discard Immediately
    else Heartbeat OK & Clean
        BL-->>ME: CLEAN
        ME->>ME: Place Order via API (Tier 2)
        ME->>DB: Log Transaction (Atomic)
    end
```

## 3. Physical Emergency Exit (Exponential Backoff)
API BAN 리스크를 피하면서도 생존을 확정 짓는 비정한 탈출 시퀀스입니다.

```mermaid
sequenceDiagram
    participant ME as Main Engine
    participant API as Broker API

    loop Check Index Data
        alt Flash Crash Triggered (-1.2% in 3min)
            ME->>ME: Set Backoff = [0.1s, 0.5s, 1.0s, 2.0s, 5.0s]
            loop For each Delay in Backoff
                ME->>API: Send IOC Order (Best Bid)
                alt Partially/Fully Filled
                    API-->>ME: Execution Success
                    ME->>ME: Exit Loop
                else Fail & Attempts < 5
                    ME->>ME: Wait (Delay)
                else Fail & Attempts == 5
                    ME->>API: Hard Market Dump (Lowest Bid)
                    ME->>ME: Force Finality
                end
            end
        end
    end
```

## 4. 15:15 Lock-Step Morphing Sequence
경합 조건을 방지하고 데이터 무결성을 보장하는 마감 전환 시퀀스입니다.

```mermaid
sequenceDiagram
    participant IE as Intra Engine
    participant SE as Swing Engine
    participant DB as Atomic DB

    Note over IE, SE: 15:15 Maintenance Window
    SE->>IE: Request Control Lock
    IE->>IE: Pause All Buy/Sell Monitoring
    IE-->>SE: Lock Granted
    
    rect rgb(240, 240, 240)
        Note right of DB: Atomic Transaction
        SE->>DB: Update Status & Snapshot Profit
        DB-->>SE: Commit Success
    end
    
    SE->>IE: Release Lock
    IE->>IE: Resume Monitoring (Wait for 15:20)
```
