# [자동매매 통합 아키텍처 설계서 - Universal Quant Core]

## 1. 설계 철학 (Philosophy)
- **Single Source of Truth**: 실전(Live)과 백테스트(Backtest)는 동일한 전략 코드(`strategy.py`)를 공유한다.
- **Physical Friction**: 백테스트는 단순 가격 체결이 아닌, 실전의 IOC 슬리피지와 지연(Latency)을 강제로 시뮬레이션한다.
- **Data Agnostic**: 전략 엔진은 데이터가 어디서 오는지(API vs DB) 상관없이 동일한 `MarketState` 인터페이스로 작동한다.

---

## 2. 핵심 모듈 구성 (Module Specs)

### A. Data Interface (DI)
- **Live Mode**: 증권사 API로부터 실시간 호가/체결 데이터를 수신하여 `MarketState` 객체 생성.
- **Backtest Mode**: Parquet/DuckDB에서 1분봉 데이터를 로드하여 `MarketState` 시계열 생성.

### B. Universal Strategy Engine (USE)
- **Input**: `MarketState` (표준화된 가격/지수/수급 데이터)
- **Params**: ATR 승수, VWAP 임계치 등 (Optimizer로부터 주입받음)
- **Output**: `SignalIntent` (Action: Buy/Sell/Hold, Weight: 0~1.0, Price: IOC/Limit)

### C. Execution & Friction Layer (EFL)
- **Live**: 실제 주문 엔진을 호출하여 증권사 전송.
- **Backtest**: `SignalIntent`를 받아 '체결 가능성' 판단. 
  - **Slip Math**: `Execution_Price = Market_Price * (1 + Friction_Factor)` (Friction_Factor는 변동성에 비례)

### D. Optimizer (Optuna)
- **Method**: Bayesian Optimization.
- **Objective**: `Maximize(Return / MDD) - Penalty(High_Frequency_Churn)`

## 3. 통합 시퀀스 다이어그램 (Universal Flow)

```mermaid
sequenceDiagram
    participant OPT as Optimizer (Optuna)
    participant DATA as Data Interface (Live/History)
    participant STRAT as Universal Strategy Engine
    participant EFL as Execution & Friction Layer
    participant PERF as Performance Monitor

    OPT->>STRAT: 1. Inject Optimized Parameters
    
    loop Each Step (Minute/Tick)
        DATA->>STRAT: 2. Provide MarketState (Standardized)
        STRAT->>STRAT: 3. Calculate Math (Sizing, VWAP, etc.)
        STRAT->>EFL: 4. Emit SignalIntent
        
        alt Mode: Backtest
            EFL->>EFL: Apply Physical Friction (Slippage)
            EFL-->>PERF: Log Simulated Trade
        else Mode: Live
            EFL->>EFL: Check API Rate-Limit (Backoff)
            EFL->>EFL: Send Actual IOC Order
            EFL-->>PERF: Log Real Transaction
        end
    end

    PERF->>OPT: 5. Return Metric (Sharpe/MDD)
    OPT->>OPT: 6. Update Bayesian Model & Repeat
```

## 4. 백테스트 안정성 보장 (Pessimistic Backtest)
1. **Gap Down Simulation**: 익일 시초가 갭하락을 역사적 최댓값으로 시뮬레이션하여 사이징의 유효성 검증.
2. **Execution Latency**: 주문 발생 후 0.5초~1초의 지연 시간을 강제로 삽입하여 '고점 추격' 리스크 확인.
3. **Blacklist Sync**: 과거 데이터상의 상폐/거래정지 종목을 완벽히 필터링하여 생존 편향(Survivorship Bias) 제거.
